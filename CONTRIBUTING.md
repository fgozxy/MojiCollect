## 🚀 快速开始

### 环境准备

1. **Fork 仓库**
   ```bash
   # 在GitHub上点击Fork按钮，然后克隆到本地
   git clone https://github.com/your-username/japanese-card-game.git
   cd japanese-card-game
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动开发服务器**
   ```bash
   npm run dev
   ```

4. **开始开发**
   - 在浏览器中打开 `http://localhost:8080`
   - 开始进行修改和测试

## 📋 开发流程

### 1. 创建分支

在开始新功能或修复之前，请创建一个新分支：

```bash
# 功能分支
git checkout -b feature/新功能名称

# 修复分支
git checkout -b fix/问题描述

# 文档分支
git checkout -b docs/文档更新
```

### 2. 开发规范

#### 代码风格
- 使用2个空格进行缩进
- 使用ES6+语法
- 函数和类使用驼峰命名法
- 常量使用大写字母和下划线
- 为关键代码添加注释

#### 代码示例
```javascript
// ✅ 好的示例
class DataManager {
    constructor() {
        this.words = [];
        this.settings = {};
    }

    /**
     * 添加新单词
     * @param {Object} wordData - 单词数据
     * @returns {boolean} 是否添加成功
     */
    addWord(wordData) {
        // 验证数据
        if (!this.validateWordData(wordData)) {
            return false;
        }

        this.words.push(wordData);
        return true;
    }
}

// ❌ 不好的示例
class datamanager{
    constructor(){
        this.words=[]
    }
    addword(data){
        this.words.push(data)
    }
}
```

#### 文件命名
- JavaScript文件使用camelCase：`gameLogic.js`
- CSS文件使用kebab-case：`style.css`
- 文档文件使用中文：`使用说明.md`

### 3. 提交规范

使用有意义的提交信息：

```bash
# 功能开发
git commit -m "feat: 添加单词导入功能"

# 问题修复
git commit -m "fix: 修复音频播放兼容性问题"

# 文档更新
git commit -m "docs: 更新README使用说明"

# 样式调整
git commit -m "style: 优化移动端显示效果"

# 代码重构
git commit -m "refactor: 重构数据管理模块"
```

## 🐛 报告问题

在报告Bug之前，请确保：

1. **检查现有Issue**：查看是否已经有人报告了相同问题
2. **提供详细信息**：
   - 问题描述
   - 复现步骤
   - 期望行为
   - 实际行为
   - 环境信息（浏览器、操作系统）
   - 截图（如适用）

### Issue模板


## 🔧 代码审查

### Pull Request流程

1. **创建Pull Request**
   - 使用清晰的标题和描述
   - 关联相关的Issue
   - 添加适当的标签

2. **代码要求**
   - 代码通过所有测试
   - 遵循项目代码规范
   - 包含必要的注释

3. **审查标准**
   - 功能正确性

   - 性能影响

   - 用户体验
